# QueryCraft Service and AI Utilization Guide

이 문서는 QueryCraft의 주요 기능을 분석가 관점에서 정리하고, 각 기능의 목적과 기대 성과, 그리고 AI 기술을 실제 업무에 접목할 수 있는 방법을 설명합니다. 프로토타입 경험을 **업무 효율 개선의 근거**로 제시할 수 있도록 작성했습니다.

## 1) 핵심 서비스 요약

QueryCraft는 학습형 SQL 분석 플랫폼으로, 다음 기능이 핵심입니다.

- PA 분석: 실무형 문제 풀이와 정답 피드백
- RCA 분석: 이상 징후/원인 분석 문제 풀이
- Practice: 무한 연습 모드
- AI Lab: 자연어→SQL, 결과 인사이트 자동 생성
- 추천: 사용자 맞춤 문제 추천
- 스케줄러: 매일 데이터 및 문제 생성

이 문서는 각 기능을 **업무 목적 → 기대 성과 → AI 활용 방식** 순서로 설명합니다.

## 2) 기능별 상세 문서

### 2.1 PA 분석 (Product Analytics)

**목적**
- 실무형 SQL 문제를 통해 분석 사고력과 실행력을 강화
- 데이터 스키마 이해, KPI 도출, 퍼널 분석 능력 검증

**기대 성과**
- 핵심 KPI 분석 역량 향상
- 문제 풀이 전환율 개선
- 풀이 시간 감소 및 정답률 향상

**AI 활용 관점**
- 문제 생성 자동화: 데이터 요약과 프롬프트 기반 문제 생성
- 정답 피드백 자동화: 제출 결과에 대한 개선 지점 제시
- 힌트 자동화: 막힌 지점에 대한 단계적 힌트 제공

**분석가 관점의 활용 이유**
- 문제 풀이 데이터를 통해 학습 병목을 발견하고 개선할 수 있음
- 문제의 난이도/토픽/전환율을 기반으로 콘텐츠 품질 관리 가능

### 2.2 RCA 분석 (Root Cause Analysis)

**목적**
- 이상 징후의 원인을 SQL로 규명하는 사고력 강화
- 실무형 장애 분석 시나리오 대응

**기대 성과**
- 비교 분석 능력 향상 (기간/세그먼트/채널 비교)
- 복합 조인/윈도우 분석 역량 강화

**AI 활용 관점**
- 시나리오 기반 문제 생성: 실제 장애 보고서 톤으로 문제 구성
- 원인 분석 논리 검증: AI가 제출 SQL의 해석 오류를 피드백

**분석가 관점의 활용 이유**
- 실제 장애 대응 프로세스를 학습하고 리포트 구조를 체득
- 실무에서 즉시 적용 가능한 분석 템플릿 확보

### 2.3 Practice (무한 연습)

**목적**
- 단일 문제 반복 풀이로 기본기 강화
- 특정 SQL 패턴 반복 연습

**기대 성과**
- 기본 문법 숙련도 증가
- 약점 주제(조인/윈도우/집계) 보완

**AI 활용 관점**
- 맞춤형 문제 추천과 난이도 조정
- 반복 풀이에서 흔히 발생하는 오류 패턴 학습

**분석가 관점의 활용 이유**
- 개인별 학습 히스토리를 기반으로 개선이 필요한 영역 파악 가능

### 2.4 AI Lab (Text-to-SQL, Insight)

**목적**
- 자연어 질문을 SQL로 변환하여 실행 속도 향상
- 결과 데이터를 비즈니스 인사이트로 변환

**기대 성과**
- 작성 시간 단축
- 분석 결과 해석 품질 향상
- 초보자의 진입 장벽 완화

**AI 활용 관점**
- Text-to-SQL: 스키마 기반 SQL 자동 생성
- AI Insight: 결과 표 요약과 액션 플랜 제안
- 분석 텍스트 자동 작성으로 보고서 작성 속도 개선

**분석가 관점의 활용 이유**
- 실제 회사 업무에서 분석 시간을 줄이고 반복 작업을 자동화
- SQL 초안 작성 및 검증 단계에서 업무 효율 상승
- 포트폴리오에 AI 기반 분석 업무 자동화 사례로 제시 가능

### 2.5 추천 (Adaptive Learning)

**목적**
- 사용자 수준에 맞는 문제 추천
- 학습 효율 최적화

**기대 성과**
- 문제 풀이 지속성 증가
- 목표 난이도 도달률 향상

**AI 활용 관점**
- 사용자 행동 데이터를 기반으로 추천 모델 개선
- 유사 문제군 자동 분류 및 노출

**분석가 관점의 활용 이유**
- 학습 퍼널과 추천 전환율을 실험/검증할 수 있음
- 데이터 기반 개인화 전략 경험을 증명 가능

### 2.6 스케줄러 (데이터/문제 생성)

**목적**
- 매일 신규 데이터와 문제를 자동 생성

**기대 성과**
- 신선한 문제 제공으로 학습 지속성 향상
- 문제 반복 노출 감소

**AI 활용 관점**
- 프롬프트 버전 관리 및 실험 그룹 분기
- 문제 생성 품질 모니터링과 회귀 탐지

**분석가 관점의 활용 이유**
- 자동 생성 시스템의 품질 지표를 설계하고 운영 가능
- A/B 실험 설계 경험 축적 가능

## 3) AI 접목 관점에서의 실무 적용 방식

### 3.1 Text-to-SQL의 실무 활용

- 요구사항을 자연어로 정리하고 초안을 자동 생성
- 복잡한 쿼리를 빠르게 만들고 수정만 수행
- 신입 분석가의 생산성을 높이는 보조 도구로 활용

### 3.2 Insight 생성의 실무 활용

- 결과표 요약 및 핵심 인사이트 자동 생성
- 리포트 초안 작성 시간 단축
- 액션 플랜 제시로 의사결정 지원

### 3.3 Prompt 버전 관리와 실험

- 문제 생성 프롬프트를 버전화하여 품질 비교 가능
- `prompt_version`, `experiment_group` 기반 성과 분석
- 실험 결과를 근거로 프롬프트 개선

### 3.4 분석 효율 개선의 증거로 활용

- AI Lab 프로토타입은 “업무 효율 개선 아이디어를 실제로 구현한 사례”로 활용 가능
- 문제 생성 자동화는 “콘텐츠 운영 자동화 경험”의 증거로 활용 가능
